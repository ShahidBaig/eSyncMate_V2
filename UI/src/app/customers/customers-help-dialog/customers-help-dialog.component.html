touch_app What You Can Do

These are the main actions available for managing your partners:

person_add
Add Customer

Create a new marketplace partner. Enter the partner name, ERP Customer ID, ISA identifiers, receiver IDs, and select the marketplace type. Once saved, the partner is ready for order and product integrations.

edit
Edit Customer

Update an existing partner's details — change the display name, ERP Customer ID, ISA identifiers, receiver IDs, or marketplace assignment. Changes take effect immediately for all future orders.

notifications_active
Alert Configuration

Set up automated email notifications for this partner. Configure alerts for order received, order errors, ASN confirmations, cancellations, and inventory sync status. Each alert can have multiple email recipients.

assignment Customer Details

Each customer record contains the following key fields:

person
Name The display name for this partner (e.g., "Knott — Mirakl")
tag
ERP Customer ID Must match the ERP system's customer code exactly (e.g., KNO8068)
fingerprint
ISA Customer ID The EDI ISA segment identifier used for electronic data interchange
receipt_long
ISA 810 / 856 Receiver ID EDI receiver IDs for invoice (810) and shipment notification (856) documents
storefront
Marketplace The marketplace platform this partner operates on (Target Plus, Amazon, etc.)

notifications_active Alert Configuration

Each customer can have automated email alerts configured for important events:

shopping_cart Order Alerts

Get notified when new orders arrive, when orders fail to sync to ERP, or when cancellation requests come in from the marketplace.

Order Received Order Error Cancellation
local_shipping Shipment & Sync Alerts

Get notified when shipment confirmations (ASN) are sent, when ASN fails, or when inventory sync completes or encounters issues.

ASN Sent ASN Error Sync Status
lightbulb Each alert type can have multiple email recipients. You can add, edit, or remove recipients at any time without affecting the customer's order processing.

storefront Understanding Marketplaces

Each customer is linked to a marketplace platform. Here are the supported marketplace types:

storeTARGET PLUS

Target's third-party marketplace. Orders arrive via Target Plus API with specific EDI requirements for acknowledgments and shipment notifications.

shopping_bagMIRAKL / KNOT

Mirakl-powered marketplaces (including Knot). Orders are pulled via Mirakl API with unique offer and order ID formats.

inventoryAMAZON SP-API

Amazon Seller Central via SP-API. Supports order sync, inventory feeds, and product listing updates through Amazon's API framework.

local_mallWALMART

Walmart Marketplace integration. Handles order retrieval, shipment confirmations, and inventory updates through Walmart's partner API.

homeLOWES

Lowe's marketplace integration. Manages home improvement product orders with specific shipping and compliance requirements.

checkroomMACY'S

Macy's marketplace integration. Handles fashion and home goods orders with department store-specific fulfillment workflows.
