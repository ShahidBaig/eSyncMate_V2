Orders — Help Guide

Everything you need to know about managing your orders in eSyncMate

route How Orders Work in eSyncMate

eSyncMate receives orders from your retail partners and manages them through each stage automatically:

storefront
Order Received Marketplace sends order
arrow_forward
thumb_up
Acknowledged eSyncMate confirms receipt
arrow_forward
sync
Synced to ERP Sales Order created
arrow_forward
local_shipping
Shipped ASN sent to marketplace
lightbulb If an order is cancelled by the marketplace or ERP, eSyncMate handles the cancellation automatically and notifies the marketplace. Orders can be fully cancelled or partially cancelled (some items cancelled, others still active).

search Finding Your Orders

Use any of these filters to quickly find the orders you need:

tag
eSyncMate ID The unique order number assigned by eSyncMate when the order is received
confirmation_number
Customer PO The Purchase Order number from the marketplace (the retailer's order reference)
receipt_long
ERP SO# The Sales Order number created in your ERP system after the order is synced
date_range
Date Range Filter orders by the date range they were received in eSyncMate
flag
Status Filter by current order status (New, Synced, Shipped, Error, etc.)
storefront
Customer Name Filter orders by retail partner (Amazon, Walmart, Target, Lowes, etc.)

label Understanding Order Statuses

Each order has a color-coded status so you can track its progress at a glance:

fiber_newNew

Order has been received from the marketplace and saved in eSyncMate. Waiting to be sent to your ERP system.

check_circleSynced

Order has been successfully sent to your ERP system. A Sales Order number has been created and confirmed.

local_shippingShipped

All items have been shipped. eSyncMate has sent the shipment notification (ASN) with tracking details to the marketplace.

splitscreenPartially Shipped

Some items have been shipped but others are still pending. eSyncMate will continue to track the remaining items.

cancelCancelled

The entire order has been cancelled. eSyncMate has notified the marketplace about the cancellation.

content_cutPartially Cancelled

Some items in the order have been cancelled, but the remaining items are still being processed or shipped.

errorError

Something went wrong while sending this order to your ERP. Use Re-Process to retry sending it.

sync_problemAsn Error

The shipment notification failed to send to the marketplace.

report_problemAck Error

Order acknowledgment failed to send to the marketplace.

touch_app What You Can Do

These action buttons are available on each order row:

visibility
View Order Details

See the complete order information including shipping address, order line items, quantities, prices, and individual line statuses. You can edit the shipping address when the order is in New or Error status.

folder_open
Show Files

View the complete processing history of the order — every data exchange between eSyncMate, your ERP system, and the marketplace is recorded here.

refresh
Re-Process Order

Available only for orders in Error status. eSyncMate will retry processing the order and sending it to your ERP.

preview Inside the Order Detail View

When you click View Order Details, you'll see two tabs:

location_on Shipping Address

View or edit the delivery address — name, company, street, city, state, zip code, and country.

lightbulb You can only edit the shipping address when the order status is New or Error.

folder_open Show Files — What Each Activity Means

eSyncMate records every data exchange for each order. Click Show Files to see the full history:

1
Order Received from Customer The original order data as it was received from the marketplace
2
Order Acknowledged in Customer Portal eSyncMate sent a confirmation back to the marketplace that the order was received
3
Order Request Sent to ERP eSyncMate sent the order data to your ERP system to create a Sales Order
4
Order Created in ERP Your ERP system confirmed the Sales Order was created successfully
5
ASN Received from ERP Shipment details (tracking number, carrier, items shipped) were received from your ERP/warehouse
6
ASN Sent to Customer eSyncMate sent the shipment notification to the marketplace so your customer gets tracking info
7
Cancellation Data If any items were cancelled — the cancellation request from ERP and the confirmation sent to the marketplace
