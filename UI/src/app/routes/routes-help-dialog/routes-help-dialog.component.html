Routes — Help Guide

Configure integration routes that move data between your systems

route How Routes Work

A route is an automated pipeline that reads data from one system, transforms it, and sends it to another:

storage
Source Reads data from ERP or API
arrow_forward
transform
Map Transforms the data
arrow_forward
storefront

touch_app What You Can Do

Use these actions to manage your routes:

add_circle
Add Route

Create a new integration route. Choose the route type, pick source and destination connectors, assign a map, and link a partner group.

edit
Edit Route

Update the route name, connectors, map, partner group, or status. Changes take effect on the next scheduled run.

history
Route Log

View the execution history for this route including timestamps, success or failure status, and any error messages.

read_more
Route Data

View the raw data that was sent and received during route execution for troubleshooting purposes.

settings Route Components

Every route is made up of these key building blocks:

cable
Source Connector Where data is read from — a database, API, or file server
cable
Destination Connector Where data is sent to — a marketplace API or your ERP
map
Map The transformation that converts data between source and destination formats
category
Route Type Determines what the route does — orders, inventory, shipments, prices, etc.
group_work
Partner Group Identifies which trading partners are involved in this data exchange

play_circle Route Execution

Routes do not run on their own. They are triggered by Flows:

lightbulb Routes define what data gets moved and how it is transformed. Flows control when routes run by assigning schedules. To activate a route, add it as a route detail inside a Flow and set the schedule to your desired frequency (minutely, hourly, daily, weekly, or monthly).
