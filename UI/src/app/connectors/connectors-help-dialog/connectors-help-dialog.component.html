category Connector Types

Each connector uses one of these communication methods:

dnsSQL Server

Connects directly to a database using stored procedures or queries. Typically used for reading inventory, orders, and product data from your ERP system.

cloudREST API

Connects to marketplace APIs over HTTP (Target Plus, Mirakl, Amazon, Walmart). Supports GET, POST, and PUT requests with headers and authentication.

folder_sharedFTP / SFTP

Transfers files to and from remote servers. Used for EDI document exchange (850, 855, 856, 810) with trading partners.

touch_app What You Can Do

Use these actions to manage your connectors:

add_circle
Add Connector

Create a new connector by choosing its type, providing a name, and configuring the connection data. You'll fill in details like the URL, headers, credentials, or connection string depending on the type.

edit
Edit Connector

Update an existing connector's name, type, or connection data. Any changes you make will take effect the next time a route using this connector runs.

settings Setting Up a Connector

When adding or editing a connector, you'll work with these configuration areas:

info General

Enter the connector name and select the connectivity type (SQL Server, REST, or FTP). This determines which additional fields you'll need to fill in.

dns SQL Server Setup

Provide the connection string to your database, specify the command type (stored procedure or query), and set the command to execute.

cloud REST API Setup

Enter the base URL for the API endpoint and configure HTTP headers including authorization keys for secure access.

tune Connection Data Fields

These are the key fields you'll configure in the connection data (JSON format):

1
ConnectivityType Specifies how the connector communicates — SqlServer, Rest, or FTP
2
BaseUrl The API endpoint address for REST connectors (e.g., https://marketplace-api.example.com)
3
Headers HTTP headers for authentication and content type — each header has a "name" and "value" pair
4
ConnectionString The database connection string used by SQL Server connectors to reach your ERP database
5
Command / CommandType The SQL command to run (stored procedure name or query) and its type (SP or QUERY)
6
CustomerID The ERP Customer ID that this connector is associated with for data filtering
lightbulb REST connectors require headers with authorization keys. SQL connectors need a valid connection string.
