touch_app What You Can Do

Use these actions to manage your alerts:

add_circle
Add Alert

Create a new alert configuration. Give it a name, assign it to a customer, set up the notification query, and add the email addresses that should receive the alert.

edit
Edit Alert

Update the alert name, customer assignment, notification query, or email recipients. Changes take effect on the next alert check.

delete
Delete Alert

Remove an alert configuration permanently. The alert will stop sending notifications immediately after deletion.

settings Alert Setup

When adding or editing an alert, you will work with these fields:

label
Alert Name A clear, descriptive name so you can easily identify what this alert monitors
code
Query The notification query that determines when this alert should fire
person
Customer Assignment Which marketplace partner this alert is linked to
email
Email Recipients The email addresses that will receive notifications when this alert fires

mail Common Alert Types

Here are some of the most frequently configured alert types:

shopping_cartOrder Received

Sends a notification when new orders are successfully imported from a marketplace.

errorOrder Error

Alerts your team when an order fails to process, so issues can be resolved quickly.

local_shippingASN Sent

Confirms when a shipment notification has been successfully sent to the marketplace.

warningASN Error

Alerts when a shipment notification fails to send, so you can investigate and retry.

inventory_2Inventory Sync

Sends a status update after an inventory upload completes, with success or failure details.

cancelCancellation

Notifies your team when an order cancellation has been processed at the marketplace.
