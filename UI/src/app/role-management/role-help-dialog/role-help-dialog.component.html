tab Understanding Roles

The Role Management screen has two tabs that let you manage roles and their permissions:

badge Roles Tab

Create, edit, and delete roles. Each role has a name, description, and active/inactive status. This is where you define the roles that will be available across the system.

assignment Menu Assignments Tab

Configure which menus each role can access. Select a role first, then check or uncheck View, Add, Edit, and Delete permissions for every menu in the system.

lock Permission Levels

Each menu can have up to four permission levels assigned per role:

visibilityView

User can see the menu and data. Without this permission, the menu is completely hidden from the user's navigation.

add_circleAdd

User can create new records. The "New" or "Add" buttons will appear on the page when this is enabled.

editEdit

User can modify existing records. The "Edit" button will appear on each row when this is enabled.

deleteDelete

User can remove records. The "Delete" button will appear on each row when this is enabled.

touch_app What You Can Do

Here are the main actions available on this screen:

add_circle
Create Role

Click New Role to create a new role. Enter a name and description, set it as active, and save. The role will then appear in the roles table and can be assigned permissions.

edit
Edit Role

Click the Edit button on any role to update its name, description, or active status. You can also deactivate a role without deleting it.

assignment
Configure Menu Access

Click the Assign Menus button on a role to open the Menu Assignments tab. From there, expand each module to see its menus and set View, Add, Edit, and Delete permissions per menu. Use Select All to quickly grant full access to an entire module.

account_tree How Permissions Flow

Here is what happens when a user with an assigned role logs in:

1
User Logs In The user enters their credentials and authenticates with the system
2
System Loads Role Permissions The system looks up the user's assigned role and loads all menu permissions for that role
3
Navigation Shows Only Permitted Menus The sidebar navigation only displays menus where the user's role has View permission enabled
4
Each Page Checks Add, Edit, and Delete Permissions When the user opens a page, the system checks which actions are allowed and shows or hides buttons accordingly
5
Buttons Appear or Hide Based on Permissions If the role does not have Add permission, the "New" button is hidden. Same for Edit and Delete buttons on each row.
lightbulb Changes to role permissions take effect the next time the user logs in.

shield Default Roles

The system comes with these pre-configured roles to get you started:

admin_panel_settingsSuperAdmin

Full system access to every menu and every permission level. This role can manage all settings, users, and configurations.

manage_accountsAdmin

Access to all menus except Role Management. Admins can manage day-to-day operations but cannot change role permissions.

engineeringOperator

Access to Orders and Products with Add and Edit permissions only. Operators can create and update records but cannot delete them.

personViewer

Read-only access to Orders and Products. Viewers can see data but cannot create, edit, or delete any records.

visibility_off Hidden Menus

lightbulb Some internal menus (Connectors, Maps, Routes, etc.) are hidden from the Role Management screen. These can only be assigned directly to specific users by an administrator.
