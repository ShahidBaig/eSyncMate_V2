Route Types — Help Guide

Categorize your integration routes by what they do

info What Route Types Are

Route types label the purpose of each route so you can quickly tell what it does:

categoryOverview

Each route type represents a specific kind of data exchange — like importing orders, uploading inventory, or sending shipment notifications. When you create a route, you pick a route type to define its function. Route types keep your integrations organized and easy to understand at a glance.

touch_app What You Can Do

Use these actions to manage your route types:

add_circle
Add Route Type

Create a new route type by giving it a name and description. The ID is automatically assigned by the system.

edit
Edit Route Type

Update the name or description of an existing route type. Routes already using this type are not affected.

list Common Route Types

Here are some of the most frequently used route types in the system:

shopping_cartOrder Import

Pulls new orders from a marketplace and brings them into your system for processing.

inventory_2Inventory Feed

Sends your warehouse stock levels to a marketplace so product availability stays up to date.

attach_moneyPrice Sync

Uploads product pricing to a marketplace to keep your listed prices current.

local_shippingASN / Shipment

Sends shipment notifications to the marketplace so they know when orders are on their way.

cancelCancellation

Sends order line cancellations to the marketplace when items cannot be fulfilled.

receipt_longPlace Order

Sends imported marketplace orders to your ERP for sales order creation.
