search How to Search Inventory Batches

To view inventory data, you must first select a customer. Here's how the search works:

1
Select ERP Customer ID Choose a marketplace partner from the searchable dropdown. This is required — no data loads until a customer is selected
2
Select Route Type (Optional) After selecting a customer, the Route Type dropdown automatically loads only the inventory route types available for that customer
3
Set Date Range (Optional) Start Date defaults to 7 days ago and Finish Date defaults to today. Adjust if needed
4
Click Search eSyncMate loads all inventory batches matching your filters into the data grid
lightbulb You must select a Customer before searching. The Route Type dropdown only shows relevant options after a customer is selected. You can also filter by Item ID or Status for more specific results.

label Batch Statuses

Each batch has a color-coded status that indicates where it is in the processing lifecycle:

hourglass_topProcessing

When set: As soon as eSyncMate begins reading inventory data from the ERP and starts processing the batch.

What it means: The batch is currently being processed. Items are being read, mapped, and prepared for sending to the marketplace. The batch is not yet complete.

What to do: Wait for the batch to finish. You can check back shortly — most batches complete within a few minutes.

check_circleCompleted

When set: After all items in the batch have been successfully processed and sent to the marketplace partner.

What it means: The inventory feed finished successfully. All items were processed and stock levels have been updated on the marketplace.

What to do: No action needed. You can click Show Inventory to verify the item-level details if required.

errorError

When set: When something goes wrong during processing — for example, a connection failure to the marketplace API, invalid data, or a timeout.

What it means: The batch did not complete successfully. Some or all items may not have been sent to the marketplace.

What to do: Check the Exceptions page for detailed error information. Contact support if the issue persists.

touch_app Show Inventory — Batch Detail Popup

Click the Show Inventory icon button on any batch row to open the batch detail popup:

info Popup Header

Shows the Customer ID, the batch Status (color-coded chip), and the total item count in the batch.

search Search by Item ID

Use the search bar at the top to filter items within the batch by Item ID. Press Enter or click Search to apply the filter. Clear the search to see all items again.

view_column Item Data Columns

Show Inventory (action button), Item ID, Sent/Received Date, Status (NEW or SYNCED), Customer Item Code, ETA Date, ETA Qty, Total ATS, and individual warehouse ATS columns (L10, L21, L28, etc.).

lightbulb The date column label changes based on the route type: it shows "Received Date" for Full and Differential inventory feeds (data received from ERP), and "Sent Date" for upload feeds (data sent to the marketplace).
lightbulb ATS means "Available to Sell" — the total quantity available across all warehouse locations. Each warehouse column (L10, L21, L28, etc.) shows the stock at that specific warehouse location.

swap_horiz Item-Level Statuses

Inside the batch detail popup, each individual item has its own status:

fiber_newNEW

When set: When the item is first received from the ERP and saved into eSyncMate.

What it means: The item has been imported into the system but has not yet been sent to the marketplace partner. It is queued and waiting to be included in the next outbound feed.

check_circleSYNCED

When set: After the item's inventory data has been successfully sent to the marketplace partner.

What it means: The item's stock levels have been successfully transmitted to the marketplace. The retailer's system now has the updated quantity for this item.
