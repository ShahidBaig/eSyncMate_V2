label Understanding User Statuses

Each user account has a status that controls their ability to access the system:

check_circleACTIVE

User can log in and access the system. They will see menus and data based on their assigned role and customers.

blockBLOCKED

User cannot log in until an admin reactivates their account. Use this to temporarily suspend access without deleting the user.

touch_app What You Can Do

These are the actions available for managing user accounts:

person_add
Add User

Creates a new user account. You will set their login ID, name, email, password, role, and assign them to one or more customers.

edit
Edit User

Modify an existing user's details, change their role, update their password, or reassign the customers they can access.

delete
Delete User

Permanently removes the user account from the system. This action cannot be undone, so use it carefully.

playlist_add_check Creating a New User

Follow these steps when adding a new user to the system:

1
Enter User ID Choose a unique login ID for the user. This is what they will use to sign in.
2
Enter First and Last Name Provide the user's full name so they can be easily identified in the system.
3
Enter Email and Phone Add contact information. The email is used for notifications and password recovery.
4
Set Password Create a secure password between 8 and 15 characters long.
5
Select Role Choose a role that determines what menus and actions the user can access.
6
Set Status Set the account to Active so the user can log in right away, or Blocked to keep the account on hold.
7
Assign Customers Select one or more customers (retail partners) the user should have access to.
lightbulb The User ID must be unique across the system. Passwords must be between 8 and 15 characters long.

admin_panel_settings User Roles

Each user is assigned a role that controls what they can see and do in the system:

securitySuperAdmin

Full access to every feature in the system, including role management, system settings, and all customer data.

admin_panel_settingsAdmin

Access to all menus and features except role management. Can manage users, customers, orders, and products.

engineeringOperator

Can view and work with orders and products. Limited to add and edit actions only — no delete permissions.

visibilityViewer

Read-only access to the system. Can view orders, products, and reports but cannot make any changes.
