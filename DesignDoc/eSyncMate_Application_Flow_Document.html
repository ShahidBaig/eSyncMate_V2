01. Application Overview

eSyncMate is an enterprise B2B Electronic Data Interchange (EDI) management platform designed for the home furnishing industry. The platform automates order processing, product catalog management, inventory tracking, and pricing operations across multiple retail partners.

Key Capabilities

✉

EDI Processing

Full EDI document lifecycle: 850 Purchase Orders, 855 Acknowledgments, 856 ASN, 810 Invoices

☷

Product Catalog

Product catalog and pricing management with bulk upload and template-driven imports

▨

Inventory Tracking

Real-time inventory feed tracking with batch processing and status monitoring

⍈

Automated Flows

Automated data flows between trading partners with configurable routes and schedules

♿

Role-Based Access

Granular RBAC with 4-level permissions and menu-level access control

☍

Multi-Language

Full internationalization support with English and Spanish language options

Role Management Interface

  • Create new roles with name, description, and active status
  • Edit existing roles
  • Delete roles

Tab 2: Menu Assignments

  • Select a role to configure
  • Module-grouped accordion with checkboxes
  • Per-menu permissions: View, Add, Edit, Delete
  • "Select All" toggle per module
  • Save button applies all changes

How Permissions Flow

Login → Role Check → Menu Rendering → Action Buttons

Hidden Menus System

🔒 Certain internal menus are permanently hidden from the Role Management UI and cannot be assigned through roles.

Hidden menus include: Connectors, Maps, Partner Groups, Routes, Route Types, Hangfire Dashboard

These menus never appear in the Roles interface, cannot be assigned through roles, and can only be granted to specific users via direct database assignment.

15. Registration (Add User)

Create new user accounts with role assignment. The registration form validates all fields before submission.

Form Fields

FieldDetails
User IDUnique identifier for the account
First Name, Last NameUser's display name
EmailContact email address
Phone NumberContact phone number
Password8–15 characters + confirmation
RoleDropdown: SuperAdmin, Admin, Operator, Viewer
StatusActive / Blocked
Customer AssignmentMulti-select for data filtering scope

Process

  1. Admin fills in user details in the registration form
  2. Selects a role from the dropdown
  3. Assigns customer access (multi-select)
  4. Clicks "Create Account"
  5. Backend creates user record + UserRoles entry
  6. New user can immediately log in
🔔

16. Notifications

Real-time event notifications for order and processing status changes, accessible from the header bell icon.

Features

  • Bell icon in header with unread count badge
  • Notification dropdown with status icons
  • Color-coded indicators: Completed Failed Running
  • Time-ago display (Just now, 5m ago, 2h ago, 3d ago)
  • Mark individual or all notifications as read
  • Server time synchronization for accurate timestamps
